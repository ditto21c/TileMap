8방향 Bitmask 그림 예시

현재 타일 X 주변 8칸을 검사해서 연결된 방향의 숫자를 더합니다. 그 합계가 Tiles By Mask 배열의 인덱스입니다.

방향별 값

128Up Left
1Up
2Up Right
64Left
XCurrent
4Right
32Down Left
16Down
8Down Right

예를 들어 위쪽 타일이 같은 그룹이면 1을 더하고, 오른쪽 타일이 같은 그룹이면 4를 더합니다.

최종 mask가 85라면 Unity Inspector의 Tiles By Mask 배열에서 Element 85에 해당 Tile asset을 넣으면 됩니다.

1 + 4 + 16 + 64 = 85

Tiles By Mask[85]에 십자형으로 연결된 Tile asset을 넣습니다.

Tiles By Mask[20]에 오른쪽과 아래가 이어지는 코너 Tile asset을 넣습니다.

Tiles By Mask[255]에는 사방이 꽉 찬 중앙 Tile asset을 넣습니다.

대각선 처리 규칙

대각선은 그냥 같은 타일이라고 바로 연결하지 않습니다. 양쪽 직교 방향이 모두 연결되어야 합니다. 그래야 모서리 모양이 깨지지 않습니다.

오른쪽 위 타일이 같아도, 위쪽이 연결되지 않으면 2는 더하지 않음

UpRight는 Up과 Right가 둘 다 연결되어 있을 때만 인정합니다. 이 예시는 오른쪽만 연결되어 있으므로 mask는 4입니다.

자주 쓰는 Mask 예시

연결 방향 계산 배열 위치 넣을 타일 모양
연결 없음 0 Tiles By Mask[0] 혼자 떨어진 섬/기둥 모양
위 + 아래 1 + 16 = 17 Tiles By Mask[17] 세로 직선
왼쪽 + 오른쪽 64 + 4 = 68 Tiles By Mask[68] 가로 직선
위 + 오른쪽 1 + 4 = 5 Tiles By Mask[5] 코너
위 + 오른쪽 + 아래 + 왼쪽 1 + 4 + 16 + 64 = 85 Tiles By Mask[85] 십자 연결
모든 방향 255 Tiles By Mask[255] 중앙/내부 타일
실제 작업에서는 모든 mask를 처음부터 채우지 말고, Fallback Tile을 먼저 넣은 뒤 게임 화면에서 자주 보이는 mask부터 하나씩 채우는 방식이 빠릅니다.
